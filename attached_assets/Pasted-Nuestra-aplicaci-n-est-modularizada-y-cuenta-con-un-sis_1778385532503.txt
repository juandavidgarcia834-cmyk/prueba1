Nuestra aplicación está modularizada y cuenta con un sistema de inicio de sesión que almacena el rol del usuario en la variable de estado (ej. st.session_state._rol_usuario). Actualmente existe una tabla en Supabase para validar el inicio de sesión. Necesitamos implementar un panel de control de usuarios exclusivo para administradores y aplicar un Control de Acceso Basado en Roles (RBAC) estricto en toda la aplicación.

Objetivos:

Crear un módulo de Gestión de Usuarios.

Proteger la visibilidad del menú lateral.

Restringir acciones de modificación/eliminación de datos en toda la app según el rol.

⚠️ REGLAS ESTRICTAS (CRÍTICO):

Cero cambios en la interfaz base: No alteres el tema visual. La aplicación debe mantener su diseño limpio, profesional y en modo claro (Light Mode) adaptado a un entorno industrial. Usa componentes estándar de Streamlit (st.dataframe, st.button, etc.).

Cero roturas funcionales: La lógica actual de cálculo y flujo de datos debe permanecer intacta.

Plan de Acción Requerido:

Paso 1: Nuevo Componente de Administración
Crea un nuevo archivo llamado components/admin_usuarios.py. Este archivo debe contener una interfaz CRUD (Crear, Leer, Actualizar, Eliminar) para manejar a los usuarios.

Debe incluir un formulario para agregar usuarios con los campos: Usuario (texto), Contraseña (texto/password) y Rol (un st.selectbox que solo permita elegir entre 'Administrador' y 'Operario').

Debe mostrar una tabla (st.dataframe) con los usuarios actuales (ocultando las contraseñas por seguridad).

Debe permitir modificar el rol o eliminar usuarios seleccionados.

Conecta estas acciones directamente a la tabla correspondiente en Supabase utilizando la conexión existente en db/supabase_client.py.

Paso 2: Modificación del Menú Lateral
Modifica el archivo components/sidebar.py.

Añade un nuevo botón o enlace de navegación llamado "Gestión de Usuarios".

Condición estricta: Este botón SOLO debe renderizarse si el rol del usuario en session_state es igual a 'Administrador'. Si es 'Operario', el botón no debe existir en la interfaz.

Paso 3: Restricción de Roles en la App
Revisa los archivos correspondientes a la visualización de datos (como el Historial o Dashboards).

Si actualmente existen botones, íconos o funciones para modificar o eliminar registros de datos de calidad o rutas, envuélvelos en una condición para que solo aparezcan si el rol es 'Administrador'.

El 'Operario' solo tiene permisos de Lectura (ver dashboards e historial) y Escritura inicial (agregar datos de rutas nuevas).

Ejecuta estos cambios paso a paso, actualizando primero el sidebar y creando el componente de administración. Confirma cuando hayas terminado para verificar que no hay errores de sintaxis.
